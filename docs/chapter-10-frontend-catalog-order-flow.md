# 第10章: Catalog/Order連携のフロント導線追加

第9章でログイン画面とOpenAPI連携を実装しました。  
第10章では、ログイン後に商品一覧からカート投入し、Order APIで注文作成までをつなぎます。

## 10-1. この章のゴール

- ログイン後に `/products` へ遷移できる
- Catalog API の商品一覧を表示できる
- カートに追加・数量変更・削除ができる
- 合計金額を表示し、Order API で注文作成できる
- UIを実務寄りに改善する（ヘッダー/アカウント/通知/配色）

## 10-2. 画面構成の整理（ページ分離）

`App.tsx` に画面ロジックを詰め込まず、ページ単位で分離します。

- `LoginPage`（ログイン）
- `ProductsPage`（商品一覧 + カート + 注文）

これにより、責務が明確になり、今後の機能追加時に保守しやすくなります。

## 10-3. Products画面の導線

- 商品一覧を `GET /products` で取得
- 商品カードからカートに追加
- カートで数量変更/削除
- 注文時は `POST /orders` を商品ごとに実行

想定内エラーはメッセージ化してUIへ返します。

- `401`: 再ログインを促す
- `403`: 権限不足
- `409`: 在庫不足

## 10-4. カートUIの改善（スライドドロワー）

カートは商品一覧下の単純セクションではなく、右側のスライドドロワーに変更しました。

- ヘッダーの `カート` ボタンで開閉
- `商品点数` / `合計金額` を常時表示
- 明細ごとに `小計` を表示
- 下部固定ボタンで注文確定

## 10-5. ヘッダー導入（実務でよくある構成）

Products画面上部に共通ヘッダーを追加しました。

- プロダクト識別（Inventory Platform / Catalog）
- 通知アイコン（未読ドット）
- アカウントアイコン
- アカウントメニュー内に `ログアウト`

これにより、画面内の重複ボタンを減らし、機能をヘッダーに集約できます。

## 10-6. 配色のリファイン

配色は、強い青系トーンから、目に優しいダークグレー基調に調整しました。

- 背景: `zinc-900` ベース
- カード/パネル: 半透明グレー
- 枠線: 薄めのグレーで主張を抑制
- 主要CTA: 白ではなくグレー寄り（`zinc-300`）

この調整で、長時間見ても疲れにくいトーンに寄せています。
